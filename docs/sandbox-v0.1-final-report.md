# Sandbox v0.1 Final Report

## Stage Goal

Deliver a stable container-level sandbox foundation for AgentRebuild with:

- a real Docker execution path for Linux VM validation
- a mock backend for Windows development and non-Docker test environments
- policy-controlled execution, workspace isolation, artifacts, and audit logging

## Implementation Scope

Implemented:

- Docker runtime provider
- Mock runtime provider
- SandboxManager
- Tool Policy Engine
- Tool security declarations
- Blocked command denylist
- Workspace isolation strategies
- Artifact collection
- Audit JSONL logging
- `sandbox:check`
- `sandbox:smoke`
- `sandbox.exec`
- Windows mock development path
- Linux Docker validation path

Excluded from v0.1:

- gVisor
- Firecracker
- Kata Containers
- true egress proxy enforcement
- session-level long-lived container reuse
- remote sandbox workers
- automatic artifact diff write-back
- full MCP side-effect containerization

## Architecture

```text
Gateway / Tool Executor
  -> Tool Policy Engine
  -> SandboxManager
  -> Docker Runtime or Mock Runtime
  -> workspace / artifacts / audit
```

## Security Strategy

- Default `network=none`
- Default `workspaceAccess=copy`
- No default `rw` mount of the real project
- No host home mount
- No `~/.ssh`, `~/.aws`, `~/.docker`, `.env`, or `docker.sock` mount
- No `--privileged`
- No `--network host`
- Docker flags generated by sandbox code, not by the model
- Env keys only in audit logs
- Blocked commands rejected before runtime execution

## Dual-Environment Strategy

Windows:

- development, editing, `typecheck`, standard test runs, mock sandbox testing
- mock backend is not a security boundary
- must return `decision="mock-sandbox"`

Linux VM:

- real Docker acceptance path
- Docker backend must return `decision="sandbox"`
- `sandbox:check` and `sandbox:smoke` are the runtime acceptance commands

## Known Boundaries

- Mock backend does not execute real commands and does not provide real isolation
- Docker validation depends on the Linux VM environment
- MCP side effects are not fully containerized in v0.1
- No outbound proxy enforcement yet

## No Further Work In v0.1

Sandbox v0.1 is complete and frozen. The project should now move on to Gateway, Memory, Tool Registry, Agent Loop, MCP, and skill-system work without continuing to expand sandbox scope in this stage.
